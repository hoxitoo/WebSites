# Сайт «Колибри» — как запустить для показа

Имиджевый новогодний лендинг ТК «Колибри». Ниже — как поднять сайт на своём
компьютере и показать заказчику. Ничего, кроме Node.js, не нужно.

## Требования (один раз)

1. Установить **Node.js 20 LTS или новее**: https://nodejs.org (кнопка "LTS").
   Проверить в терминале: `node -v` → должно показать v20.x или выше.
2. Получить код:
   ```bash
   git clone https://github.com/hoxitoo/WebSites.git
   cd WebSites/Calibri_gift
   npm install
   ```
   (если код уже есть на диске — просто `cd` в папку `Calibri_gift`)

## Показ заказчику — продакшен-режим (рекомендуется)

Быстрый, без дев-оверлеев, максимально плавные анимации:

```bash
npm run build
npm run start
```

Открыть **http://localhost:3000** — готово. Остановить сервер: `Ctrl+C`.

> `npm run build` нужно повторять после каждого изменения кода.
> Для разработки удобнее dev-режим (см. ниже) — там всё обновляется само.

## Показать на телефоне (та же Wi-Fi сеть)

```bash
npm run start -- -H 0.0.0.0
```

Узнать IP компьютера: `ipconfig` (Windows) → строка IPv4, например `192.168.1.42`.
На телефоне открыть `http://192.168.1.42:3000`.
Скролл-сцену с подарком обязательно смотреть и на телефоне!

## Показать по ссылке (заказчик не рядом) — ngrok

1. Запустить сайт в прод-режиме (первый терминал):
   ```bash
   npm run build
   npm run start
   ```
2. Во втором терминале поднять туннель:
   ```bash
   ngrok http 3000
   ```
3. Скопировать из вывода ngrok ссылку `Forwarding` вида
   `https://xxxx-xx-xx.ngrok-free.app` — отправить заказчику.

Нюансы:
- Ссылка живёт, пока работают **оба** терминала (сервер + ngrok).
- На бесплатном тарифе ngrok при каждом запуске даёт новую ссылку, а перед
  сайтом показывает страницу-предупреждение с кнопкой «Visit Site» —
  предупредить заказчика, что это нормально, нажать кнопку один раз.
- Постоянный адрес (без смены ссылки): в личном кабинете ngrok создать
  бесплатный static domain и запускать
  `ngrok http 3000 --url=ваш-домен.ngrok-free.app`.
- ngrok не установлен/не залогинен: https://ngrok.com/download, затем
  `ngrok config add-authtoken <токен из кабинета>`.

Запасной вариант без ngrok: `npx localtunnel --port 3000`.
Постоянная демо-ссылка: деплой на Vercel (`npm i -g vercel && vercel`).

## Режим разработки

```bash
npm run dev
```

http://localhost:3000, изменения кода подхватываются на лету.

## Куда падают заявки с формы

- Без настройки: просто пишутся в консоль сервера (режим разработки).
- Боевой режим (Google Таблица + письмо менеджеру): настроить по инструкции
  [docs/lead-setup.md](docs/lead-setup.md) и указать `LEAD_WEBHOOK_URL`
  в файле `.env.local` (пример — в `.env.example`).
